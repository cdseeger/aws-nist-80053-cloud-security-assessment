# AWS NIST SP 800-53 Cloud Security Control Assessment

This project demonstrates a Governance, Risk, and Compliance (GRC) security assessment performed on a fictional federal contractor operating in an AWS cloud environment.

## Organization
Acme Federal Services (Fictional)

## Environment
Amazon Web Services (AWS)

## Data Sensitivity
- Personally Identifiable Information (PII)
- Controlled Unclassified Information (CUI)

## Framework
NIST SP 800-53 Security Controls

## Controls Evaluated
- AC-2 – Account Management
- AC-6 – Least Privilege
- IA-2 – Identification & Authentication (MFA)
- AU-6 – Log Review
- CM-2 – Baseline Configuration
- SC-7 – Boundary Protection

## Project Components
This repository includes:

- Control Assessment (NIST SP 800-53)
- Risk Register
- Executive Risk Summary
- Access Control Policy

## Objective
The purpose of this assessment is to evaluate whether security controls in the AWS environment are implemented effectively and to identify gaps that may expose the organization to security risks.

## Key Findings
The assessment identified several security gaps including:

- Lack of MFA enforcement for all AWS IAM users
- No formal user access review process
- Insufficient log monitoring procedures
- Absence of documented baseline configuration standards
